# EUDM动作序列组合数量详解

## 🎯 快速回答

**加上横向后的总组合数：27个**（不是243个）

## 📊 详细解释

### 1. 动作类型

**纵向动作** (Longitudinal)：
- M = Maintain（保持）
- A = Accelerate（加速）
- D = Decelerate（减速）

**横向动作** (Lateral)：
- K = Lane Keeping（保持车道）
- L = Lane Change Left（左变道）
- R = Lane Change Right（右变道）

### 2. 时间结构

```
树高度: 5层
每层时长: 1秒
总规划时域: 5秒
```

### 3. ⚠️ 关键约束

**纵向动作在5秒内保持不变！**

代码设计中，纵向动作一旦选定就保持整个5秒：
```cpp
for (int lon = 0; lon < 3; lon++) {  // 选择一个纵向模式
    // 后续5秒都使用相同的纵向动作
    for (int h = 1; h < tree_height_; ++h) {
        ongoing_action_seq.push_back(
            DcpAction(DcpLonAction(lon), ...));  // 同一个lon
    }
}
```

### 4. 纵向模式（3种）

```
模式1: [MMMMM] - 5秒内全部保持速度
模式2: [AAAAA] - 5秒内全部加速
模式3: [DDDDD] - 5秒内全部减速
```

**不会有以下混合序列**：
- ❌ [AAMMD] - 前2秒加速，中间保持，最后减速
- ❌ [MADMA] - 交替的纵向动作
- ❌ [AAADD] - 先加速后减速

### 5. 横向变化（每种纵向模式9种）

以纵向模式 `[AAAAA]` 为例，横向有9种变化：

| 序号 | 横向序列 | 说明 | 开始变道时间 |
|-----|---------|------|-------------|
| 1 | KLLLLL | 第2秒开始左变道，持续4秒 | t=1s |
| 2 | KRRRR | 第2秒开始右变道，持续4秒 | t=1s |
| 3 | KKLLL | 第3秒开始左变道，持续3秒 | t=2s |
| 4 | KKRRR | 第3秒开始右变道，持续3秒 | t=2s |
| 5 | KKKLL | 第4秒开始左变道，持续2秒 | t=3s |
| 6 | KKKRR | 第4秒开始右变道，持续2秒 | t=3s |
| 7 | KKKKL | 第5秒开始左变道，持续1秒 | t=4s |
| 8 | KKKKR | 第5秒开始右变道，持续1秒 | t=4s |
| 9 | KKKKK | 全程保持车道 | - |

### 6. 总组合计算

```
纵向模式：3种 (M全程/A全程/D全程)
横向变化：9种 (每种纵向模式)
────────────────────────────
总计：3 × 9 = 27个序列
```

### 8. 为什么不是3^5 = 243？

如果每层都独立选择纵向和横向动作，理论上有：
```
每层：3纵向 × 3横向 = 9种
5层：9^5 = 59,049种
```

但这样会导致：
- ❌ 搜索空间爆炸（59,049个序列）
- ❌ 纵向动作频繁切换（不稳定）
- ❌ 行为不连贯（M→A→D→M）
- ❌ 计算时间过长

### 9. 设计优势

通过约束纵向动作保持不变：

✅ **搜索效率**：59,049 → 27（减少2187倍）  
✅ **行为连贯**：纵向策略明确  
✅ **计算可行**：27个线程，<50ms  
✅ **仍然灵活**：横向9种时机选择  
✅ **平滑控制**：IDM提供连续加减速

### 11. 总结表格

| 维度 | 理论值 | 实际值 | 约束原因 |
|-----|--------|--------|---------|
| 纵向动作类型 | 3 | 3 | 无约束 |
| 纵向时间自由度 | 3^5=243 | 3 | 全程保持同一动作 |
| 横向动作类型 | 3 | 3 | 无约束 |
| 横向时间自由度 | 多种 | 9 | 一次变道，选择时机 |
| **总序列数** | **59,049** | **27** | **工程平衡** |

## 💡 关键要点

1. **纵向只有3种模式**：全M、全A、全D
2. **横向有9种时机**：何时开始变道（或不变道）
3. **总共27个序列**：3 × 9 = 27
4. **不是243或59049**：有约束，为了效率和稳定性
5. **仍能灵活决策**：27个序列覆盖了主要场景
